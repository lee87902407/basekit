# AGENTS

本文件定义 `basekit` 仓库的 AI 协作与文档维护规范。

## 语言规范

1. 代码注释默认使用中文。
2. 说明性文档、开发规范、示例文档默认使用中文。
3. 对外英文材料需要同步维护，但不能替代中文主说明。

## README 规范

1. `README.md` 是统一入口，负责汇总导航，不承载过多细节。
2. `README.zh-CN.md` 作为中文主说明。
3. `README.en.md` 作为英文说明。
4. 新增主模块、变更模块能力或调整模块结构时，必须同步更新上述 README 文件。

## 模块新增规范

每新增一个新的主模块，必须在同一次变更中完成以下事项：

1. 新增或更新模块实现代码。
2. 更新 `README.md`、`README.zh-CN.md`、`README.en.md` 中对应入口。
3. 新增模块示例文档：`docs/examples/<module>.md`。
4. 新增或更新模块示例代码：`examples/<module>/`。
5. 如有 API、约束、边界条件或限制，必须写入模块文档与示例文档。

## 示例文档规范

模块示例文档建议包含以下内容：

1. 模块用途。
2. 适用场景。
3. 最小示例。
4. 常见注意事项。
5. 与 README 的跳转关系。

## 提交要求

1. 不要只改代码不改文档。
2. 不要新增模块但缺少示例文档或示例代码。
3. 文档链接必须可从 `README.md` 跳转访问。

## Go 代码兼容性规则

1. 在实现代码中，禁止使用 `for _, v := range ...` 这类 value 变量绑定风格来遍历可能被重复绑定或被误捕获的对象集合。
2. 尤其是在需要兼容低版本 Go、闭包、延迟执行或后续可能扩展为引用语义的代码中，统一改用 `for i := range ...` 再通过索引访问。
3. 如无充分理由，实现代码优先使用索引访问写法，避免产生这类兼容性隐患。
